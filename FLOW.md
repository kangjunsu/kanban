## 5. 이벤트 흐름 상세

### 5.1 이벤트 리스너 맵
| DOM 요소 | 이벤트 | 핸들러 함수 | 목적 |
|---------|--------|-------------|------|
| `.card` | `dragstart` | `handleDragStart()` | 드래그 시작, 카드 참조 저장 |
| `.card` | `dragend` | `handleDragEnd()` | 드래그 종료, 정리 작업 |
| `.column` | `dragover` | `handleDragOver()` | 드롭 허용 (preventDefault) |
| `.column` | `dragenter` | `handleDragEnter()` | 컬럼 진입, 시각 피드백 추가 |
| `.column` | `dragleave` | `handleDragLeave()` | 컬럼 이탈, 시각 피드백 제거 |
| `.column` | `drop` | `handleDrop()` | 카드 이동 실행 |
| `#addCardBtn` | `click` | `addNewCard()` | 새 카드 추가 프로세스 시작 |

### 5.2 이벤트 전파 차단
```
drop 이벤트 발생
    ↓
e.stopPropagation() 호출
    ↓
이벤트 버블링 중단
    ↓
상위 요소로 전파 방지
```

## 7. 에러 및 예외 처리 흐름

### 7.1 카드 추가 취소 흐름
```
"새 카드 추가" 버튼 클릭
        ↓

```

        ↓
dragend 이벤트 자동 발생
        ↓
카드가 원래 위치로 복귀
        ↓
.dragging, .drag-over 클래스 제거

## 10. 사용자 시나리오별 플로우 요약

### 시나리오 1: 하루 일과 시작
```
페이지 접속 → 기존 카드 확인 → 새 작업 추가 (여러 개) → 작업 시작
```

### 시나리오 2: 작업 진행 중
```
작업 시작 → TO-DO 카드를 IN-PROGRESS로 이동 → 작업 진행
```

### 시나리오 3: 작업 완료
```
작업 완료 → IN-PROGRESS 카드를 DONE으로 이동 → 성취감 획득
```

### 시나리오 4: 팀 회의
```
화면 공유 → 각 컬럼 확인 → 진행도 논의 → 필요 시 카드 추가/이동
```

## 11. 향후 확장 플로우 (Phase 2)

### 11.1 LocalStorage 통합 플로우
```
카드 추가/이동/삭제 이벤트
        ↓
DOM 업데이트
        ↓
카드 데이터를 JSON으로 직렬화
        ↓
localStorage.setItem('kanbanCards', json)
        ↓
페이지 새로고침
        ↓
localStorage.getItem('kanbanCards')
        ↓
JSON 파싱 후 DOM에 카드 재생성
```

### 11.2 백엔드 연동 플로우
```
카드 추가/이동/삭제 이벤트
        ↓
Optimistic UI Update (즉시 UI 업데이트)
        ↓
fetch() API 호출 (POST/PUT/DELETE)
        ↓
서버 응답 대기
        ↓
┌─────────┬─────────┐
│ 성공    │  실패   │
│         │         │
↓         ↓         
UI 유지   UI 롤백 + 에러 메시지
```
