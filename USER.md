# User Personas & User Stories
# 칸반보드 애플리케이션

## 1. User Personas

### Persona 1: 민수 (개인 개발자)
**배경**
- 나이: 28세
- 직업: 프리랜서 웹 개발자
- 기술 수준: 중급
- 근무 환경: 재택근무

**목표 및 동기**
- 다수의 프로젝트를 동시에 진행하며 작업 우선순위 관리 필요
- 단순하고 빠르게 사용할 수 있는 도구 선호
- 복잡한 설정이나 학습 곡선이 낮은 툴을 원함

**페인 포인트**
- Trello, Jira 같은 도구는 너무 무겁고 기능이 과함
- 간단한 TODO 관리만 필요한데 로그인, 설정이 번거로움
- 빠르게 브라우저에서 열고 바로 사용하고 싶음

**사용 시나리오**
- 아침에 하루 계획을 세우며 작업 카드 생성
- 작업 진행 중 상태를 TO-DO → IN-PROGRESS로 변경
- 작업 완료 시 DONE으로 이동하며 성취감 획득

**기대 가치**
- 즉시 사용 가능한 간편함
- 시각적으로 작업 진행도 파악
- 별도의 회원가입 없이 바로 사용

---

### Persona 2: 지혜 (스타트업 팀장)
**배경**
- 나이: 32세
- 직업: 스타트업 개발팀 리드
- 기술 수준: 고급
- 팀 규모: 5명

**목표 및 동기**
- 팀원들의 작업 현황을 한눈에 파악
- 스프린트 단위로 작업 관리
- 빠르고 직관적인 협업 도구 필요

**페인 포인트**
- 기존 칸반 도구들은 초기 설정이 복잡함
- 팀원들이 도구 사용법 익히는 데 시간 소요
- 실시간 동기화가 필요한데 기존 도구는 느림

**사용 시나리오**
- 주간 회의에서 새 스프린트 작업을 카드로 생성
- 팀원들이 각자 작업을 IN-PROGRESS로 이동
- 데일리 스탠드업에서 칸반보드를 화면에 띄워 공유
- 완료된 작업을 DONE으로 옮기며 진행도 추적

**기대 가치**
- 팀 전체의 작업 가시성 확보
- 빠른 학습 곡선으로 팀원 온보딩 간소화
- 향후 다중 사용자 기능 추가 시 팀 협업 강화

---

### Persona 3: 영희 (디자이너)
**배경**
- 나이: 26세
- 직업: UI/UX 디자이너
- 기술 수준: 초급 (비개발자)
- 근무 환경: 회사

**목표 및 동기**
- 디자인 프로젝트의 작업 단계별 관리
- 복잡한 기술 지식 없이 사용 가능한 툴
- 예쁘고 직관적인 인터페이스 선호

**페인 포인트**
- 기술 용어나 복잡한 설정 창은 이해하기 어려움
- 키보드 단축키나 복잡한 조작은 불편함
- 마우스로 간단히 드래그 앤 드롭하는 것이 편함

**사용 시나리오**
- 디자인 프로젝트의 단계별 작업을 카드로 생성 (와이어프레임, 목업, 프로토타입 등)
- 작업 진행에 따라 카드를 이동
- 클라이언트에게 진행 상황 공유 시 칸반보드 캡처

**기대 가치**
- 아름답고 직관적인 UI
- 별도의 학습 없이 바로 사용
- 시각적으로 작업 흐름 파악

## 2. User Stories

### Epic 1: 칸반보드 기본 사용
**As a** 사용자  
**I want to** 칸반보드를 열어서 현재 작업 현황을 한눈에 보고 싶다  
**So that** 내 작업의 전체적인 흐름을 빠르게 파악할 수 있다

#### Story 1.1: 칸반보드 뷰 확인
- **As a** 사용자
- **I want to** 웹페이지를 열면 TO-DO, IN-PROGRESS, DONE 컬럼이 보인다
- **So that** 작업 단계를 명확히 인식할 수 있다
- **Acceptance Criteria**:
  - [ ] 3개의 컬럼이 가로로 배치되어 있다
  - [ ] 각 컬럼에 제목이 명확히 표시된다
  - [ ] 각 컬럼의 카드 개수가 숫자로 표시된다

#### Story 1.2: 초기 샘플 카드 확인
- **As a** 새 사용자
- **I want to** 각 컬럼에 샘플 카드가 미리 있다
- **So that** 어떤 식으로 사용하는지 바로 이해할 수 있다
- **Acceptance Criteria**:
  - [ ] TO-DO에 3개, IN-PROGRESS에 2개, DONE에 2개의 샘플 카드 존재
  - [ ] 각 카드에 제목과 설명이 있다

---

### Epic 2: 카드 추가
**As a** 사용자  
**I want to** 새로운 작업 카드를 추가하고 싶다  
**So that** 새로운 할 일을 칸반보드에 기록할 수 있다

#### Story 2.1: 새 카드 추가 버튼 클릭
- **As a** 사용자
- **I want to** "새 카드 추가" 버튼을 클릭한다
- **So that** 새 카드 생성 프로세스를 시작할 수 있다
- **Acceptance Criteria**:
  - [ ] 화면 하단에 "새 카드 추가" 버튼이 명확히 보인다
  - [ ] 버튼에 호버 시 시각적 피드백이 있다

#### Story 2.2: 카드 정보 입력
- **As a** 사용자
- **I want to** 카드 제목과 설명을 입력한다
- **So that** 작업 내용을 명확히 기록할 수 있다
- **Acceptance Criteria**:
  - [ ] 제목 입력 프롬프트가 표시된다
  - [ ] 설명 입력 프롬프트가 표시된다
  - [ ] 제목 없이 취소하면 카드가 생성되지 않는다

#### Story 2.3: 카드가 TO-DO 컬럼에 추가됨
- **As a** 사용자
- **I want to** 새 카드가 자동으로 TO-DO 컬럼에 생성된다
- **So that** 새 작업은 항상 할 일 목록에서 시작된다
- **Acceptance Criteria**:
  - [ ] 새 카드가 TO-DO 컬럼 하단에 추가된다
  - [ ] TO-DO 컬럼의 카드 개수가 +1 증가한다

---

### Epic 3: 카드 드래그 앤 드롭
**As a** 사용자  
**I want to** 카드를 드래그하여 다른 컬럼으로 이동하고 싶다  
**So that** 작업 상태를 쉽게 변경할 수 있다

#### Story 3.1: 카드 드래그 시작
- **As a** 사용자
- **I want to** 카드를 마우스로 클릭하고 드래그한다
- **So that** 카드를 이동할 준비를 한다
- **Acceptance Criteria**:
  - [ ] 카드를 드래그하면 반투명해진다
  - [ ] 마우스 커서가 이동 커서로 변경된다
  - [ ] 카드가 약간 회전된 시각 효과가 나타난다

#### Story 3.2: 드래그 중 시각적 피드백
- **As a** 사용자
- **I want to** 카드를 다른 컬럼 위로 이동 시 시각적 피드백이 있다
- **So that** 어디에 드롭할 수 있는지 명확히 안다
- **Acceptance Criteria**:
  - [ ] 대상 컬럼의 배경색이 변경된다
  - [ ] 대상 컬럼에 점선 테두리가 표시된다
  - [ ] 컬럼에서 벗어나면 피드백이 사라진다

#### Story 3.3: 카드 드롭
- **As a** 사용자
- **I want to** 카드를 대상 컬럼에서 마우스를 놓는다
- **So that** 카드가 새 컬럼으로 이동된다
- **Acceptance Criteria**:
  - [ ] 카드가 대상 컬럼에 추가된다
  - [ ] 원래 컬럼에서 카드가 사라진다
  - [ ] 양쪽 컬럼의 카드 개수가 업데이트된다
  - [ ] 드래그 시각 효과가 제거된다

#### Story 3.4: 작업 진행 상태 변경 (TO-DO → IN-PROGRESS)
- **As a** 사용자
- **I want to** TO-DO 카드를 IN-PROGRESS로 이동한다
- **So that** 작업을 시작했음을 표시한다
- **Acceptance Criteria**:
  - [ ] 카드가 IN-PROGRESS 컬럼에 나타난다
  - [ ] 카드 내용이 그대로 유지된다

#### Story 3.5: 작업 완료 (IN-PROGRESS → DONE)
- **As a** 사용자
- **I want to** IN-PROGRESS 카드를 DONE으로 이동한다
- **So that** 작업 완료를 시각적으로 표시한다
- **Acceptance Criteria**:
  - [ ] 카드가 DONE 컬럼에 나타난다
  - [ ] DONE 컬럼의 카드 개수가 증가한다

---

### Epic 4: 반응형 디자인
**As a** 모바일 사용자  
**I want to** 모바일 기기에서도 칸반보드를 사용하고 싶다  
**So that** 언제 어디서나 작업 관리가 가능하다

#### Story 4.1: 모바일 레이아웃
- **As a** 모바일 사용자
- **I want to** 컬럼이 세로로 배치된다
- **So that** 작은 화면에서도 편하게 볼 수 있다
- **Acceptance Criteria**:
  - [ ] 768px 이하에서 컬럼이 세로로 변경된다
  - [ ] 스크롤로 모든 컬럼을 확인할 수 있다
  - [ ] 터치로 드래그 앤 드롭이 가능하다 (향후 개선)

---

### Epic 5: 카드 개수 표시
**As a** 사용자  
**I want to** 각 컬럼의 카드 개수를 실시간으로 보고 싶다  
**So that** 각 단계에 얼마나 많은 작업이 있는지 파악할 수 있다

#### Story 5.1: 실시간 카드 개수 업데이트
- **As a** 사용자
- **I want to** 카드를 추가하거나 이동할 때마다 개수가 자동으로 업데이트된다
- **So that** 별도의 새로고침 없이 정확한 개수를 본다
- **Acceptance Criteria**:
  - [ ] 카드 추가 시 해당 컬럼 개수 +1
  - [ ] 카드 이동 시 원래 컬럼 -1, 대상 컬럼 +1
  - [ ] 개수가 배지 형태로 시각적으로 강조된다

## 3. Use Cases

### Use Case 1: 일일 작업 계획 세우기
**Actor**: 민수 (개인 개발자)

**Preconditions**:
- 브라우저에서 칸반보드 페이지가 열려 있음

**Main Flow**:
1. 민수가 "새 카드 추가" 버튼을 클릭한다
2. 제목 입력 프롬프트에 "백엔드 API 구현"을 입력한다
3. 설명 입력 프롬프트에 "회원가입 API 엔드포인트 개발"을 입력한다
4. 카드가 TO-DO 컬럼에 생성된다
5. 2-4번을 반복하여 여러 작업 카드를 추가한다

**Postconditions**:
- 오늘 할 작업들이 TO-DO 컬럼에 모두 추가됨
- 각 카드에 명확한 제목과 설명이 있음

---

### Use Case 2: 작업 진행 상태 업데이트
**Actor**: 민수 (개인 개발자)

**Preconditions**:
- TO-DO 컬럼에 "백엔드 API 구현" 카드가 있음

**Main Flow**:
1. 민수가 작업을 시작하기로 결정한다
2. "백엔드 API 구현" 카드를 마우스로 클릭하고 드래그한다
3. IN-PROGRESS 컬럼 위로 카드를 이동한다
4. 컬럼의 배경색이 변경되어 드롭 가능함을 표시한다
5. 마우스를 놓는다 (드롭)
6. 카드가 IN-PROGRESS 컬럼에 나타난다
7. TO-DO 카드 개수가 감소하고 IN-PROGRESS 개수가 증가한다

**Postconditions**:
- 카드가 IN-PROGRESS 컬럼에 있음
- 시각적으로 작업 중임을 확인할 수 있음

---

### Use Case 3: 팀 미팅에서 진행도 공유
**Actor**: 지혜 (팀장)

**Preconditions**:
- 칸반보드에 팀의 작업 카드들이 각 컬럼에 분산되어 있음

**Main Flow**:
1. 지혜가 미팅 시작 시 칸반보드를 화면에 띄운다
2. TO-DO 컬럼을 가리키며 아직 시작 안 된 작업들을 확인한다
3. IN-PROGRESS 컬럼을 보며 현재 진행 중인 작업과 담당자를 언급한다
4. DONE 컬럼을 보며 이번 주 완료된 작업들을 팀과 공유한다
5. 새로운 작업을 논의하고 "새 카드 추가"로 TO-DO에 추가한다

**Postconditions**:
- 팀 전체가 현재 상황을 명확히 이해함
- 새 작업이 TO-DO 컬럼에 추가됨

## 5. Feature Prioritization (MoSCoW)

### Must Have (필수)
- ✅ 3개 컬럼 칸반보드 뷰
- ✅ 카드 드래그 앤 드롭
- ✅ 새 카드 추가
- ✅ 카드 개수 표시
- ✅ 반응형 디자인

### Should Have (중요)
- ⏳ 카드 편집 기능
- ⏳ 카드 삭제 기능
- ⏳ LocalStorage 데이터 영속성

### Could Have (있으면 좋음)
- 카드 우선순위 (High/Medium/Low)
- 카드 태그/라벨
- 검색 및 필터링
- 다크 모드

### Won't Have (이번엔 제외)
- 다중 사용자 실시간 동기화
- 사용자 인증
- 알림 기능
- 모바일 앱
